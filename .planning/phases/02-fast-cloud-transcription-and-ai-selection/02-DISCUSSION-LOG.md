## Discussion Summary

We aligned on the integration details for the transcription and AI selection workflows. The goal is to offload heavy operations to APIs (Groq and Gemini) for free, fast, and scalable cloud execution.

## Areas Discussed

### 1. Handling 1-Hour Audios on Groq Whisper API
- **Problem:** Groq restricts uploads to 25MB. A raw 1-hour `.wav` file is hundreds of MB.
- **Solution:** Compress the extracted audio to a low-bitrate MP3 mono file (32kbps, 16kHz) using FFmpeg. This compresses the file to ~10MB, fitting within limits.
- **Fallback:** If Groq is unavailable, fallback to local `faster-whisper` on CPU.

### 2. Selection AI Engine
- **Decision:** Switch to `gemini-1.5-pro` for candidates selection.
- **Rationale:** Better reasoning and hook understanding. The 50 requests/day free cota easily accommodates the user's 5 videos/day.

### 3. Token & Performance Optimization for Gemini
- **Problem:** Sending 600+ candidates generated from a 1-hour video to Gemini uses large prompt tokens and overflows output token limits (8192 max output tokens).
- **Solution:** Run the fast local keyword analyzer first to score all candidates, sort them, and send only the **top 40** candidates to Gemini for deep evaluation. This guarantees fast responses, fits limits, and preserves accuracy.
